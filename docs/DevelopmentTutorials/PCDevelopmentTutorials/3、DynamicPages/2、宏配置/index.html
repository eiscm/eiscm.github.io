宏配置以及使用

TMS 运输管理系统-宏定义

对于前端来讲，可以简单理解为，使用同一个接口，通过传入不同的参数来完成不同的功能

接口调用​

  • 接口URL： /htms/v1/${getCurrentOrganizationId()}/tp/macro/excuteMacro
  • 接口 method: POST
  • 该接口的封装于 tms-front/src/services/api 文件里的 transferByGrand 方法

接口传参格式

{
  paramIn: {
    // 单据的 idList
    list: [
      { id: "" }
    ],
    // 其他参数
    // ...
  },
  // 宏代码
  macroCode: "xxx"
}

统一处理返回结果

封装了统一处理返回结果提示的组件 MacroNotification;

组件的引入

// 注：tmsCommon 引入简写请在 alias 文件中处理，指的是 tms-front/src/common
import MacroNotification from 'tmsCommon/htms/components/MacroNotification';

使用

// 引入
import { transferByGrand } from 'tms-front/lib/services/api';
import MacroNotification from 'tmsCommon/htms/components/MacroNotification';

// 使用
transferByGrand({
  paramIn: {
    list: [{id: 1}, {id: 2}]
  },
  macroCode: 'XXX'
}).then(res=>{
  MacroNotification.open({
    ...res,
    // 成功回调
    onSuccess: ()=>{},
    // 失败回调，在关闭失败弹框时触发
    errorClose: ()=>{},
  });
})

宏定义​

上面提到的参数 macroCode 指的是宏定义里的宏代码，宏定义的菜单路径为 【流程引擎】--【流程配置】--【宏定义】

可以在页面中查对应宏的入参。

注：宏里面配置的「对象」，使用的时候需要对应，打个比方，TMS_OM_SAVE 是订单保存的宏，对象为订单，代表这个宏只能针对订单去使用，传参中的 paramIn 的 list 就应该为订单的 idList。

具体可在这里查看宏定义的介绍。

开发​

可以使用宏dataSet: MacroDataSet，继承自 c7n/pro DataSet，只是集成了宏的统一接口调用和返回内容的处理，和正常的dataSet一样使用，回调动作中集成 MacroNotification，对 submit 方法做了处理

import MacroDataSet from 'tmsCommon/MacroDataSet';
import {listDs} from '../store/demoDs';
import { RecordStatus } from 'choerodon-ui/pro/lib/data-set/enum';

// ......
// 实例化ds
const tableDs=new MacroDataSet(listDs)
<Table columns={columns} dataSet={tableDs} buttons={buttons}/>
// ......

// 1.批量勾选后调用宏提交
//使用macroSubmit调用批量数据的宏调用
//isSelect 传true,默认为true,可不传
  handelCancelOrder=()=>{
    const config={
      macroCode: XXX,
      onSuccess: (res) => {
        //doSomeThing
      },
    };
    this.tableDs.macroSubmit(config,true);
  }

//2.行按钮调用宏
//传入该行record
//isSelect 传false
  handelCancelBooking=async (record)=>{
    const config={
      macroCode: XXXX,
      record,
      onSuccess: () => { 
        //doSomething
        this.tableDs.query();
      },
    }   
    this.tableDs.macroSubmit(config,false);
  }
//3.表单类型的保存动作调用宏
  const formDs=new MacroDataSet({...ds配置对象})
  handelSaveOrder=()=>{
    const config={
      params:...formDs.current.toData(),//获取表单数据对象
      macroCode: XXXX,
      onSuccess: (res) => {//可以拿到接口回调的内容
        //doSomeThing
      },
    };
    this.formDs.macroSubmit(config,false);
  }

也可以在页面上手动调用 request，与别的接口调用方式一致，用 MacroNotification 处理返回结果即可
