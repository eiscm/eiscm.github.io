业务组件

1、业务组件的使用​

【列表配置】添加按钮时可以选择【业务组件按钮】

2、业务组件的分类​

业务组件分类：我的组件和公共组件；其实都是【业务组件维护】中的数据，我的组件为本租户下的数据，公共组件为平台租户下的数据。

  • 我的组件：指的是本租户下【业务组件维护】中状态为「发布」的组件
  • 公共组件：指的是产品开发的标准组件，在平台租户-租户管理员角色下【业务组件维护】中状态为「发布」的组件

3、业务组件的开发​

菜单路径：【流程引擎】-【页面配置】-【业务组件维护】

PS：如果需要基于产品标准组件进行改动，把平台层对应的代码复制到租户层，进行改动。不要直接在平台层修改，否则升级时会出现问题。如果是项目需求的组件，请直接在项目租户层进行开发。

3.1 头字段介绍​

字段名说明
代码组件唯一编码
操作类型行按钮/批量按钮/独立按钮
描述该组件的业务描述
对象有值时，只有对应对象的列表才可使用；为空时，所有的列表都可以使用
组件名称组件名称
组件预览预览图片URL
代码块组件代码逻辑，代码块支持 class 组件和 function 组件，但组件命名必须为 APP

3.2 行字段「变量维护」​

这里维护的变量可以直接在代码块中使用，可以理解为组件的入参。

3.3 代码块的开发​

业务组件是使用在线编译的方式，所以是需要提前挂载依赖，产品目前引入的依赖如下，暂不支持其他依赖，如有需求，可以联系产品组添加。

开发注意事项：

  • 1、业务组件的 Modal 弹框是已经封装好的，我们这里只是开发弹框内容的部分，Modal 是由内容撑开的，所以可以通过设置内容的宽度来撑开外部 Modal。

  • 2、可以使用 function 组件，也可以使用 class 组件，但是必须命名为 App

  • 3、在组件里可以通过 props 获取列表的数据，字段如下

    const {ds, objectType, record, records, targetModal} = props;
    字段说明
    ds列表dataSet
    objectType列表对象
    record操作类型为行按钮时，当前行的record
    records表格被选中的数据，records
    targetModal组件外部Modal实例

  • 4、在组件里可以获取变量（行字段）

    `${testAttr}`

  • 5、Modal 如果有底部按钮需求，需要在代码块里写 Button，绑定事件。代码块内部能够获取到 targetModal 实例和列表 ds 实例，所以可以在代码块里关闭弹框以及触发表格刷新。

3.4 组件调试​

在组件编辑界面，发布后是可以继续编辑保存的。调试的时候不需要先【停用】。

  • 组件发布
  • 列表配置选到对应组件
  • 需要调试的时候，直接在组件编辑界面保存(若从列表无法进入编辑页面，进入查看页面后修改路由中的 view 为 edit)
  • 修改完成后，切到列表页面后，就是最新的代码了
